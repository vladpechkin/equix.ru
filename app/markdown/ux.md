# UX и DX

В создании и использовании приложений с графическим интерфейсом выделяют три понятия:
1. User Interface (UI, Интерфейс Пользователя);
2. User Experience (UX, Опыт Пользователя);
3. Developer Experience (DX, Опыт Разработчика).

User Interface - визуальная составляющая интерфейса, то, что пользователь видит. В зависимости от качества реализации, UI может быть привлекательным для пользователей или наоборот отталкивающим. Это касается цветовой схемы, шрифтов, иллюстраций, размеров и расположения элементов. Именно потому, что UI движется вперед, новые сервисы кажутся нам очень привлекательными визуально, а приложения из 2000-х - ужасно некрасивыми. 
Чтобы не мешать кастомизации сервисов в процессе создания (чтобы все приложения, созданные с помощью EQUIX, не выглядели идентично), наш фреймворк реализует UI по минимуму. Шрифты и прочие аспекты интерфейсов остаются на ваше усмотрение.

User Experience — это совокупность эмоций и впечатлений пользователя при взаимодействии с продуктом или услугой. Хороший UX содействует увеличению удовлетворенности клиентов, лояльности и, как следствие, росту доходов компании. Ключевые аспекты UX включают удобство использования, доступность и эффективность. Плохой UX - это когда пользователь не может найти ту функцию или страницу, которая ему нужна, не понимает, где кнопки, что делает каждая из них и куда ведет та или иная ссылка. EQUIX изначально задуман как преимущественно UX-фреймворк. Все компоненты и решения продуманы на основе научных исследований в области использования компьютеров людьми. Цвета контрастны, шрифты достаточно большие, элементы на странице не могут "слипнуться" и так далее. Наша цель - чтобы ваши пользователи наслаждались вашим продуктом.
 
Developer Experience охватывает ощущения разработчиков по отношению к инструментам и технологическому стеку, с которым они работают. Улучшение DX позволяет увеличить производительность команды, снизить время для разработки и улучшить качество кода. Плохой DX - когда разработчики пользуются устаревшими технологиями, не договариваются об общих установках и решениях, которые должны соблюдать все без исключения. EQUIX создан разработчиками для разработчиков, поэтому:
1. Использует самые современные и простые в овладении даже с нуля технологии (TailwindCSS, React и т.д.);
2. Понижает порог входа, предоставляя решения и компоненты, которые автоматически подстраиваются и становятся тем, чем должны быть, не заставляя вас думать о том, кнопкой или ссылкой, диалогом или аккордеоном делать тот или иной элемент. Пример - умные компоненты Box, Bar, View.

Инвестирование в User Experience и Developer Experience является ключевым фактором успеха современных компаний. Создавая удобные для пользователей продукты и выгодные условия для разработчиков, компании могут добиться высокой конкурентоспособности и роста.
